# Task Brief

Use this when a request is fuzzy, cross-cutting, risky, or likely to grow.

## Goal

What outcome should be true when this is done?

## Context

Relevant files, call paths, tests, configs, stack traces, screenshots, docs, prior decisions, or production facts.

## Constraints

Repo instructions, architecture rules, public contracts, compatibility requirements, performance limits, safety constraints, deployment constraints.

## Done when

Concrete acceptance conditions:

- Behavior change:
- Regression coverage:
- Cleanup complete:
- User-visible or API-visible result:

## Non-goals

What should stay untouched even if nearby code looks tempting?

## Checks

Narrow checks first, then broader checks proportional to risk.

## Risks and Tradeoffs

What is uncertain, what could regress, and which simpler alternative should be considered?
